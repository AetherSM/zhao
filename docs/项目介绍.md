# 项目介绍：辅导机构管理系统（B/S）

## 1. 项目背景

辅导机构日常运营涉及学员、教师、课程、排课、收费、考勤与统计等信息。传统方式依赖表格/人工记录，容易出错且效率低。本系统通过信息化方式实现统一管理与数据分析，提升运营效率与数据准确性。

## 2. 系统目标

- 覆盖机构核心业务闭环：**人（学员/教师）- 课（课程/报名/排课/考勤）- 钱（缴费/欠费/营收）**
- 支持多角色登录与接口级权限控制
- 提供在线接口文档（Swagger/OpenAPI）与可用测试数据，便于开发/联调/部署

## 3. 技术架构

### 3.1 后端

- Spring Boot 2.7.18
- MyBatis-Plus（CRUD + 分页 + 逻辑删除）
- MySQL 8.0
- Spring Security + JWT（无 Session）
- SpringDoc OpenAPI（Swagger UI）

### 3.2 前端

- Vue 3 + Vite
- Element Plus
- Pinia（状态管理）
- Axios（请求封装、自动注入 Token）
- ECharts（统计图表）

## 4. 功能模块

- **用户登录**：JWT 登录，前端持久化 Token
- **学员管理**：新增/编辑/删除/分页查询
- **教师管理**：新增/编辑/删除/分页查询
- **课程分类/课程管理**：分类与课程信息维护
- **课程报名**：学员报名课程（避免重复报名）
- **排课管理**：新增排课、列表查询、冲突检测
- **缴费管理**：缴费记录维护、欠费统计
- **考勤管理**：学员考勤记录维护
- **统计分析**：学员分布、课程报名统计、营收统计（按日/月/年）

## 5. 权限模型（接口级 RBAC）

角色编码（来自数据库 `sys_role.role_code`）：

- `SUPER_ADMIN`：超级管理员（全权限）
- `ORG_ADMIN`：机构管理员（业务全模块）
- `TEACHER`：教师（排课/考勤模块）
- `FINANCE`：财务（缴费/营收统计模块）

后端权限控制位置：`src/main/java/org/example/zhao/security/SecurityConfig.java`

## 6. 接口文档

- 业务测试数据（学员/教师/课程/报名/排课/缴费/考勤）
