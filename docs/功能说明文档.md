# 辅导机构管理系统 - 功能说明文档

本系统在原有基础之上，针对辅导机构的实际业务需求，进行了深度的角色差异化设计和功能扩展。

## 1. 角色与权限体系

系统目前支持五大角色，每个角色拥有完全不同的界面入口和功能权限：

- **系统管理员 (SUPER_ADMIN)**: 拥有全系统最高权限，可管理所有用户、角色、业务数据及系统配置。
- **机构管理员 (ORG_ADMIN)**: 负责机构日常运营，管理学员、教师、课程、排课及全局统计。
- **财务管理 (FINANCE)**: 专注财务流程，拥有缴费录入、欠费统计及营收报表查看权限。
- **任课教师 (TEACHER)**: 
  - 查看个人课表，管理自身相关的排课。
  - 为名下课程登记考勤。
  - **[新增]** 发布学习资源（课件、资料）及布置课后作业。
- **学生/家长 (STUDENT)**: 
  - 注册与登录专属入口。
  - 查询个人排课记录。
  - 查看个人考勤状态。
  - **[新增]** 在线查看老师发布的作业与学习资料。
  - **[新增]** 对已修课程和任课老师进行匿名/实名评价反馈。

## 2. 核心新增功能介绍

### 2.1 差异化登录与自主注册
- **双入口设计**: 登录页区分“教职工”与“学生/家长”入口，提升用户体验。
- **自主注册与自动绑定**: 
  - 支持教师和学生自主注册。
  - **关键改进**: 系统现在会在注册时**自动创建**对应的“教师档案”或“学生档案”，并完成账号与档案的**自动绑定**。这解决了新用户注册后无法发布资源或查看数据的权限空洞问题。

### 2.2 学习资源与作业中心
- **教师端**: 教师可以针对特定课程发布“作业布置”或“学习资料”，支持文本描述及外部资源链接（如云盘链接）。
- **表单优化**: 所有 ID 录入项均已改为**下拉选择框**（课程、教师等），极大降低了操作门槛。
- **管理员权限**: 管理员现在也可以发布全局资源，系统支持 `teacher_id` 为空的情况。

### 2.3 课程评价反馈系统
- **闭环反馈**: 学生在修完课程后，可以对课程质量、老师教学水平进行星级评价和文字反馈。
- **操作简化**: 评价时通过下拉列表选择课程和老师，无需手动记忆 ID。

## 3. 技术实现亮点
- **SQL 自动化部署**: 所有业务表（如评价反馈、资源中心）及各角色演示账号均已集成至 `schema.sql`。您只需在数据库中执行该脚本，即可获得一个包含完整演示数据的系统。
- **深层数据隔离**: 建立了 `sys_user_student` 和 `sys_user_teacher` 绑定体系，确保教师只能看到自己的班级，学生只能看到自己的课表。
- **权限安全**: 通过 Spring Security 细化了每一个 API 的访问权限，确保财务数据、管理功能对无关角色完全不可见。
- **开箱即用**: 系统预置了 `admin` (管理员), `teacher1` (教师), `finance1` (财务), `student1` (学生) 等多种角色的演示账号，方便快速体验差异化功能。
