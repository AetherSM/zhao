# HashMap原理

通过Key值计算hash值

hashmap是基于hash表的双列集合,jdk1.7前是数组➕链表,1.8之后,数组➕链表➕红黑树

对key调用hashcode计算hash值,再做二次hash,减少冲突

通过 `(n - 1) & hash` 计算数组下标（n 是数组长度）。

如果该位置**没有元素**，直接插入。

如果**有元素**（哈希冲突），遍历链表 / 红黑树：

- key 已存在 → 覆盖 value

- key 不存在 → 追加到链表 / 红黑树

  

JDK1.8：链表长度 ≥ 8 **且**数组长度 ≥ 64 → 链表**转为红黑树**；树节点 ≤ 6 → 退化为链表。

**扩容机制**

- 默认负载因子 `0.75`
- 扩容阈值 = 容量 × 负载因子
- 达到阈值自动**2 倍扩容**，重新计算所有元素位置
- JDK1.8 优化：扩容时不需要重新计算 hash，直接用 `原位置` 或 `原位置+旧容量`

**哈希冲突解决**

采用**链地址法**：相同下标元素挂在同一个链表 / 红黑树上。

**线程安全性**

**HashMap 是非线程安全的**。

- 1.7 多线程扩容会出现**环形链表、数据丢失**

- 1.8 修复了环形链表，但仍线程不安全

  多线程用 

  ```
  ConcurrentHashMap
  ```

  。
