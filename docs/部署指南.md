# 辅导机构管理系统 - 服务器部署指南 (端口隔离方案)

本文档根据您提供的最新 Nginx 配置，指导您如何通过**独立的端口**来部署“辅导机构管理系统”，使其与服务器上原有的项目完全隔离。

## 1. 部署架构

- **旧项目 (商城)**: 继续在 `80` 端口运行，访问地址 `http://43.156.248.36/`。
- **新项目 (辅导机构)**: 启用一个全新的 `8082` 端口进行访问。
  - **前端访问**: `http://43.156.248.36:8082/`
  - **后端服务**: 运行在 `8081` 端口，并带有 `/demo` 上下文路径。
  - **Nginx 代理**: 监听 `8082` 端口，并将所有 `/api/` 请求转发到 `http://127.0.0.1:8081/demo/api/`。

## 3. 部署步骤

### 步骤 3.1: 重新打包项目

由于代码已更新，请务必在您的开发电脑上**重新打包**前后端项目。

- **后端打包** (在项目根目录 `/Users/develop/final_college/zhao` 执行):
  ```bash
  ./mvnw clean package -DskipTests
  ```
  打包产物：`target/zhao-0.0.1-SNAPSHOT.jar`

- **前端打包** (在 `/Users/develop/final_college/zhao/frontend` 目录执行):
  ```bash
  npm install
  npm run build
  ```
  打包产物：`frontend/dist/` 整个文件夹

### 步骤 3.2: 上传文件到服务器

1.  **上传后端 jar 包**:
    ```bash
    # 在您的开发电脑上执行
    scp /Users/develop/final_college/zhao/target/zhao-0.0.1-SNAPSHOT.jar your_user@your_server_ip:/home/yrf/
    ```
    *(注意：根据 Nginx 配置，后端文件似乎没有放在 `/home/yrf/new_frontend`，您可以根据实际情况调整路径)*

2.  **上传前端 dist 文件夹内容**:
    ```bash
    # 在您的开发电脑上执行
    # 注意是上传 dist 文件夹内的所有内容，而不是 dist 文件夹本身
    scp -r /Users/develop/final_college/zhao/frontend/dist/* your_user@your_server_ip:/home/yrf/new_frontend/
    ```

### 步骤 3.3: 更新 Nginx 配置

将“豆包”提供给您的那份**完整 Nginx 配置**覆盖到您服务器的 `/etc/nginx/nginx.conf` (或宝塔面板对应的站点配置文件中)。

然后，重载 Nginx 服务：
```bash
# 在服务器上执行
sudo nginx -t
sudo systemctl reload nginx
```

### 步骤 3.4: 启动后端服务

在服务器上，进入 `jar` 包所在的目录并启动服务。

```bash
# 在服务器上执行
cd /home/yrf/

# 以后台模式运行 jar 包
nohup java -jar zhao-0.0.1-SNAPSHOT.jar > tms.log 2>&1 &
```

### 步骤 3.5: 检查防火墙/安全组

请确保您服务器的防火墙或云服务商（如阿里云、腾讯云）的安全组规则已经**开放了 `8082` 端口**的入站访问，否则外部无法访问。
